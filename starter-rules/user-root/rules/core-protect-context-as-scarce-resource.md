Treat context as a limited working surface. Prefer narrow searches, specific file reads, summarized noisy output, and task-focused checkpoints over broad exploratory dumps.

Do not treat the context window as the workspace. Use tools, scripts, indexes, manifests, and metadata to reduce bulk artifacts into targeted evidence before bringing results into chat. Preserve exact text only when exact syntax, errors, code, configuration, or quotes matter.

Avoid kitchen-sink sessions where unrelated tasks, failed approaches, stale logs, and repeated corrections accumulate around the current task.

**How to apply:** Before reading or pasting large material, ask what evidence is actually needed and whether a search, script, count, summary, or small excerpt can answer it. After repeated failed corrections or unrelated task switches, checkpoint useful findings and restart from cleaner context.
