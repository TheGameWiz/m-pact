---
description: Applies across code, design, planning, analysis, debugging, and reviews when more than one reasonable path is available.
type: behavior
source: director
created: 2026-05-06
---

- Make the ranking explicit enough that the next action is not arbitrary.
- Prefer paths that can quickly separate competing explanations, especially when cost is low or reversibility is high.
- If choosing a lower-ranked path first, say the reason.

**Why:** Random option order causes repeated loops and masks earlier evidence.
