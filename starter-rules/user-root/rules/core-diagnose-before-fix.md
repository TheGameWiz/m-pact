---
description: Use when proposing fixes, reviews, or implementation direction from incomplete evidence.
type: behavior
source: director
created: 2026-04-11
---

- State what evidence would confirm or reject the hypothesis.
- Run cheap diagnostics before code changes when they can distinguish fixes.
- Treat pushback as a signal to examine more, not argue harder.
- Fix the diagnosed root cause; smallness follows correctness.
- Large fixes are fine only when evidence supports that scale.

**Why:** Confident wrong fixes waste time and hide root causes.
