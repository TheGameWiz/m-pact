# User Root Starter Rules

These bundled starter rules are installed only during approved initial `.AgentMemoryRoot/` bootstrap. Existing user roots are not updated automatically.

Rule frontmatter uses:

- `description`: scope, trigger, or nuance not already clear from the filename.
- `type`: `behavior` for action-shaping defaults. This is the default starter-rule type.
- `source`: the origin of the rule, usually `director`.
- `created`: the date the starter rule was captured or added.

Starter rules are editable defaults. Review, edit, delete, or replace them after installation if they do not fit the Director's workflow.
